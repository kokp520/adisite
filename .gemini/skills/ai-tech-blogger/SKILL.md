# AI Tech Blogger 寫作規範

當被要求生成技術文章時，請嚴格遵守以下指導原則：

## 1. 文章屬性與格式 (Frontmatter)
- **Weight**: 必須設定為 `2` (確保其排序在 Weight 1 的「購物網站」項目之下)。
- **Date/Lastmod**: 使用當前時間 (ISO 8601 格式)。
- **Draft**: 設定為 `false`。
- **Categories**: 優先使用 `["Tech"]` 或 `["AI"]`。
- **Tags**: 根據主題提供 5-8 個具體標籤。
- **TOC**: `enable: true`。

## 2. 內容要求
- **字數限制**: 全文必須達到 **1500 字以上** 的深度內容。
- **標題設計**: 標題必須具有吸引力且專業 (例如：從 "RAG 教學" 改為 "超越檢索：深度解析 Agentic RAG 的自我修正機制與實踐")。
- **結構豐富**: 必須包含：
    - 引人入勝的導言 (Intro)。
    - 具體的技術原理分析。
    - **具體的實作方向或程式碼範例** (使用 Go, Python 或 Mermaid 圖表)。
    - 未來趨勢或挑戰。
    - 總結 (Summary)。
- **語言風格**: 繁體中文為主，專業術語使用英文，語氣專業且富有洞察力。

## 3. 輸出規範
- 只輸出 Markdown 原始碼。
- **嚴禁** 包含 ` ```markdown ` 這種外層 code block 標記。
- 必須包含 `<!--more-->` 標記於導言之後。
